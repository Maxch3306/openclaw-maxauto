# Milestone v1.0: MVP Settings

**Status:** SHIPPED 2026-03-14
**Phases:** 1-9
**Total Plans:** 10

## Overview

First milestone delivering all core settings functionality: config infrastructure, skills management (discovery, control, installation), workspace management (defaults, per-agent), and Telegram channel management (bot setup, access control, agent binding).

## Phases

### Phase 1: Config Infrastructure
**Goal**: All config writes use merge semantics, eliminating race conditions between UI sections
**Plans**: 2 plans (01-01: config.patch migration, 01-02: stale config reads fix)
**Completed**: 2026-03-14

### Phase 2: Skills Discovery
**Goal**: Users can browse all available skills and understand why any skill is unavailable
**Plans**: 1 plan (02-01: skills card grid UI)
**Completed**: 2026-03-14

### Phase 3: Skills Control
**Goal**: Users can enable/disable skills and configure API keys
**Plans**: 1 plan (03-01: toggle switch + API key input)
**Completed**: 2026-03-14

### Phase 4: Skills Installation
**Goal**: Users can install missing skill dependencies from the UI
**Plans**: 1 plan (04-01: install button + gateway skills.install)
**Completed**: 2026-03-14

### Phase 5: Workspace Defaults
**Goal**: Users can view, change, and open the default workspace directory
**Plans**: 1 plan (05-01: WorkspaceSection + dialog plugin)
**Completed**: 2026-03-14

### Phase 6: Per-Agent Workspace
**Goal**: Users can assign different workspace directories to individual agents
**Plans**: 1 plan (06-01: edit dialog folder picker + settings list)
**Completed**: 2026-03-14

### Phase 7: Telegram Bot Setup
**Goal**: Users can validate bot tokens and see connection status
**Plans**: 1 plan (07-01: token validation + status display)
**Completed**: 2026-03-14

### Phase 8: Telegram Access Control
**Goal**: Users can configure DM and group allow-lists with tag-style inputs
**Plans**: 1 plan (08-01: TagInput component + tag-based allow-lists)
**Completed**: 2026-03-14

### Phase 9: Channel-Agent Binding
**Goal**: Users can bind a Telegram bot to a specific agent
**Plans**: 1 plan (09-01: agent binding dropdown)
**Completed**: 2026-03-14

## Milestone Summary

**Key Decisions:**
- config.patch merge semantics over config.set full replacement
- Skills card grid with expand-on-click (not list or table)
- TagInput chip-style for access lists (not comma-separated text)
- @tauri-apps/plugin-dialog for native folder picker
- 1:1 agent-bot binding (not N:M)

**Key Accomplishments:**
1. Config infrastructure with merge-patch semantics eliminates race conditions
2. Skills management with 52-skill discovery, toggle, API key, and install
3. Workspace management with native folder picker and per-agent overrides
4. Telegram channel management with token validation, status display, access control, and agent binding
