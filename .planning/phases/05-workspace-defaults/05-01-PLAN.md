  <action>
Create `src/components/settings/WorkspaceSection.tsx` following the layout pattern from `GeneralSection.tsx`:

**Layout:**
- `max-w-2xl mx-auto p-6` container (same as GeneralSection)
- `h1` title: "Workspace"
- Section with `h2`: "Default Workspace"
- Card with `p-4 rounded-lg bg-[var(--color-surface)] border border-[var(--color-border)]` styling
- Brief description text (text-xs text-[var(--color-text-muted)]): explain that the workspace is the directory where agents read and write files
- Current path displayed in a `font-mono text-sm` span, truncated with `truncate` class
- Two action buttons in a flex row:
  - "Change" button -- uses `FolderOpen` icon from lucide-react, calls `open({ directory: true, title: "Select Workspace Directory" })` from `@tauri-apps/plugin-dialog`
  - "Open in Explorer" button (or "Open in Finder" on macOS) -- uses `ExternalLink` icon from lucide-react, calls `openUrl(workspacePath)` from `tauri-commands.ts`

**Button styling:** Match the GeneralSection button pattern: `flex items-center gap-1.5 text-sm px-3 py-1.5 rounded-lg border border-[var(--color-border)] text-[var(--color-text-muted)] hover:text-[var(--color-text)] hover:bg-[var(--color-surface-hover)] transition-colors`

**State management:**
- Use local React state (useState) for workspace path and loading/saving states. No Zustand store needed -- this is a simple read-display-write flow.
- On mount (useEffect), call `gateway.request("config.get", {})` to read `config.agents?.defaults?.workspace` and set it as current path. Default fallback: `"~/.openclaw-maxauto/workspace"`.
- On Change click: call `open({ directory: true, title: "Select Workspace Directory" })`. If result is null (cancelled), do nothing. If result is a valid path, call `patchConfig({ agents: { defaults: { workspace: selected } } })` then `waitForReconnect()`, then re-read config to update displayed path.
- Add a `saving` loading state -- disable both buttons while saving, show "Saving..." text on the Change button.
- Error handling: wrap patchConfig in try/catch, log errors to console.

**Non-existent directory handling (per user decision):** After folder picker returns a path, before applying, check if the path looks valid (the dialog plugin returns real filesystem paths from the native picker, so non-existence is extremely unlikely with a folder picker -- skip the confirmation for picker-selected paths since native OS dialogs only let you select existing directories).

**Wire into SettingsPage.tsx:**
- Add import: `import { WorkspaceSection } from "../components/settings/WorkspaceSection";`
- Add case in `renderSection` switch: `case "workspace": return <WorkspaceSection />;`
