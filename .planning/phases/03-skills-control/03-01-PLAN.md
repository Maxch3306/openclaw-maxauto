<task type="auto">
  <name>Task 1: Add toggle switch and optimistic toggle logic</name>
  <files>src/components/settings/SkillsSection.tsx, src/components/settings/skills-utils.ts</files>
  <action>
1. In `skills-utils.ts`, add two helper functions:
   - `skillNeedsApiKey(skill: SkillStatusEntry): boolean` -- returns `Boolean(skill.primaryEnv)`
   - `hasApiKeySet(skill: SkillStatusEntry): boolean` -- returns `Boolean(skill.primaryEnv && !skill.missing.env.includes(skill.primaryEnv))`
   - `isToggleDisabled(skill: SkillStatusEntry): boolean` -- returns true when the skill is NOT user-disabled AND NOT eligible (requirements not met, so toggle should be grayed out). A user-disabled skill always allows toggling (to re-enable). Formula: `!skill.disabled && !skill.eligible`

2. In `SkillsSection.tsx`, create a `ToggleSwitch` component (inline in same file):
   - Pure Tailwind toggle: `<button role="switch" aria-checked={checked}>` with a sliding circle
   - Styling: `h-5 w-9 rounded-full` track, `h-3.5 w-3.5 rounded-full bg-white` knob
   - ON state: `bg-[var(--color-success)]`, knob `translate-x-4`
   - OFF state: `bg-[var(--color-text-muted)]/30`, knob `translate-x-0.5`
   - Disabled state: `opacity-40 cursor-not-allowed`
   - Must call `e.stopPropagation()` to prevent card expand/collapse

3. Add toggle to the SkillCard compact view (top-right, between the status badge and chevron):
   - The toggle checked state is `!skill.disabled` (disabled=false means ON)
   - Disable toggle when `isToggleDisabled(skill)` returns true (unavailable, non-user-disabled)
   - Always-on skills (`skill.always`) should also have toggle disabled

4. Add state to `SkillsSection`:
   - `skillErrors: Record<string, string>` -- per-skill error messages
   - `busySkills: Set<string>` -- skills currently being toggled (to disable toggle during request)

5. Implement `handleToggle(skill: SkillStatusEntry)` in SkillsSection:
   - Optimistic update: immediately flip `skill.disabled` in `report.skills` array via `setReport`
   - Clear any existing error for this skill
   - Call `gateway.request("skills.update", { skillKey: skill.skillKey, enabled: skill.disabled })` -- note: if skill was disabled, `enabled: true` (flip)
   - On success: call `loadSkills()` to refresh full status
   - On error: revert the optimistic update by restoring original `disabled` value, set error message in `skillErrors[skill.skillKey]`

6. Show inline error text below the toggle when `skillErrors[skill.skillKey]` exists:
   - Small red text `text-[10px] text-[var(--color-error)]` below the card header row
   - Auto-dismiss after 5 seconds using setTimeout

7. Update the SkillCard `onToggle` prop: rename to `onCardClick` for clarity. Add new `onSkillToggle` prop for the toggle switch. Pass both from SkillsSection.
  </action>

  </verify>
