Create `src/components/settings/skills-utils.ts` with:

1. **TypeScript types** matching the gateway `skills.status` response:

```typescript
type SkillStatusEntry = {
  name: string;
  description: string;
  source: string;            // "openclaw-bundled" | "openclaw-managed" | "openclaw-workspace" | "openclaw-extra"
  bundled: boolean;
  filePath: string;          // internal, not displayed
  baseDir: string;           // internal, not displayed
  skillKey: string;          // unique identifier
  primaryEnv?: string;       // e.g. "GOOGLE_API_KEY"
  emoji?: string;            // e.g. "magnifying_glass" or unicode
  homepage?: string;
  always: boolean;
  disabled: boolean;
  blockedByAllowlist: boolean;
  eligible: boolean;         // pre-computed by gateway
  requirements: { bins: string[]; anyBins?: string[]; env: string[]; config: string[]; os: string[] };
  missing: { bins: string[]; anyBins?: string[]; env: string[]; config: string[]; os: string[] };
  configChecks: Array<{ path: string; satisfied: boolean }>;
  install: Array<{ id: string; kind: string; label: string; bins: string[] }>;
};

type SkillStatusReport = {
  workspaceDir: string;
  managedSkillsDir: string;
  skills: SkillStatusEntry[];
};

type SkillGroup = { id: string; label: string; skills: SkillStatusEntry[] };
```

Note: `anyBins` is optional because the UI type in OpenClaw omits it -- handle gracefully if absent.

2. **`groupSkills(skills: SkillStatusEntry[]): SkillGroup[]`** -- Port from openclaw/ui/src/ui/views/skills-grouping.ts. Groups by source with these categories in order: Workspace Skills (openclaw-workspace), Built-in Skills (openclaw-bundled), Installed Skills (openclaw-managed), Extra Skills (openclaw-extra). IMPORTANT: Check `skill.bundled` flag first for Built-in group (some bundled skills may have a different source). Skills matching no group go into "Other Skills" at the end. Filter out empty groups.

3. **`getSkillDisplayStatus(skill: SkillStatusEntry): "enabled" | "disabled" | "unavailable"`** -- If `skill.disabled` return "disabled". If `!skill.eligible` return "unavailable". Otherwise return "enabled".

4. **`computeSkillMissing(skill: SkillStatusEntry): string[]`** -- Returns human-readable list of missing items: `missing.bins` prefixed with "bin: ", `missing.anyBins` (if present) prefixed with "bin: " and suffixed with " (any)", `missing.env` prefixed with "env: ", `missing.config` prefixed with "config: ", `missing.os` prefixed with "os: ". Returns empty array if nothing missing.

Export all types and functions.

**Part A: Create `src/components/settings/SkillsSection.tsx`**

Follow existing patterns from IMChannelsSection.tsx (local useState/useEffect, gateway.request on mount, no props).

**State:**
- `loading: boolean` (true initially)
- `error: string | null`
- `report: SkillStatusReport | null`
- `expanded: string | null` (skillKey of currently expanded card, or null)

**Data loading:**
- Check `useAppStore(s => s.gatewayConnected)` -- if false, render disconnected state (do NOT attempt API call). See Pitfall 3 in RESEARCH.md.
- On mount (and when gatewayConnected becomes true), call `gateway.request<SkillStatusReport>("skills.status", {})`.
- Set loading/error/report states in try/catch/finally.

**Render states (in priority order):**

1. **Gateway disconnected** (`!gatewayConnected`): Centered message with Lucide `WifiOff` icon. Text: "Gateway not connected" with muted subtext "Skills information is available when the gateway is running." Per user decision: connection error message with reconnect hint.

2. **Loading** (`loading && !report`): Centered spinner (use Lucide `Loader2` with `animate-spin` class) with text "Loading skills..." below it. Per user decision: simple centered spinner, no skeleton cards.

3. **Error** (`error && !report`): Centered error message with red styling, show error text, include a "Retry" button that calls loadSkills() again.

4. **Skills grid** (has report): Group skills using `groupSkills(report.skills)`. For each non-empty group, render:

   **Category header:** Bold text with group label (e.g., "Built-in Skills"), as a section divider. Show skill count per category in muted text (Claude's discretion).

   **Card grid:** Responsive grid -- `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-3`. Per user decision: 2-3 cards per row.

   **Each skill card (collapsed):**
   - Container: `bg-[var(--color-surface)] border border-[var(--color-border)] rounded-lg p-3 cursor-pointer hover:bg-[var(--color-surface-hover)] transition-colors`
   - If unavailable: add `opacity-60` to container (per user decision: visually dimmed/muted)
   - Top row: emoji/icon (left), skill name (bold, truncate), status badge (right)
     - Emoji: If `skill.emoji` exists, try rendering as-is (may be unicode). If it looks like a shortcode (contains only word chars and underscores), fall back to Lucide `BookOpen` icon. Simple approach, no emoji library.
     - Status badge: Small rounded pill --
       - Enabled: `bg-[var(--color-success)]/20 text-[var(--color-success)]` text "Enabled"
       - Disabled: `bg-[var(--color-text-muted)]/20 text-[var(--color-text-muted)]` text "Disabled"
       - Unavailable: `bg-[var(--color-warning)]/20 text-[var(--color-warning)]` text "Unavailable"
   - Second row: 1-line description in muted text, truncated with `line-clamp-1`
   - Click handler: toggle expanded state (`setExpanded(expanded === skill.skillKey ? null : skill.skillKey)`)

   **Each skill card (expanded)** -- same card but with additional detail section below:
   - Smooth expand: use `max-height` transition or simple conditional render (Claude's discretion on animation style)
   - Divider line
   - Full description (not truncated)
   - If unavailable, show "Why unavailable" section:
     - Call `computeSkillMissing(skill)` and render each item as a monospace chip/tag (`font-mono text-xs bg-[var(--color-bg)] px-2 py-0.5 rounded`)
     - Group visually: missing dependencies (bins), missing API keys (env), system requirements (config, os)
   - If skill has `install` options: show "Install options" section with each `install[].label` as text
   - If skill has `homepage`: show link (Lucide `ExternalLink` icon) opening in default browser
