   - `allBindings: BindingEntry[]` -- full bindings array from config (to preserve non-telegram entries)
   Define a local `BindingEntry` interface: `{ type?: string; agentId: string; comment?: string; match: { channel: string; [key: string]: unknown } }`

3. **Read agents from store:** `const agents = useChatStore(s => s.agents);`

4. **Update loadTelegramConfig()** to also read `bindings[]` from the config response:
   - Extract `result.config` (already available as `cfg`)
   - Read `bindings` from `cfg` (it's at the root level, not under `channels`): `const bindings = (cfg as any).bindings ?? [];`
   - Find telegram binding: `const tgBinding = bindings.find((b: BindingEntry) => b.match?.channel === "telegram");`
   - Set `boundAgentId` to `tgBinding?.agentId ?? null`
   - Set `allBindings` to the full array

5. **Add Agent Binding dropdown in the JSX** after the Bot Token field (line ~444, after the BotFather hint paragraph) and before the DM Policy section. Per user decision: placed after bot token/status, before access control lists. Use the same select styling as dmPolicy/groupPolicy dropdowns:
   ```tsx
   {/* Agent Binding */}
   <div>
     <label className="block text-xs font-medium text-[var(--color-text-muted)] mb-1.5">
       Agent
     </label>
     <select
       value={boundAgentId ?? ""}
       onChange={(e) => setBoundAgentId(e.target.value || null)}
       className="w-full px-3 py-2 rounded-lg bg-[var(--color-bg)] border border-[var(--color-border)] text-sm text-[var(--color-text)] focus:outline-none focus:border-[var(--color-accent)]"
     >
       <option value="">Select an agent...</option>
       {agents.map((a) => (
         <option key={a.agentId} value={a.agentId}>
           {a.emoji ? `${a.emoji} ` : ""}{a.name}
         </option>
       ))}
     </select>
     {/* Warning if bound agent no longer exists */}
     {boundAgentId && !agents.some(a => a.agentId === boundAgentId) && (
       <div className="flex items-center gap-1.5 mt-1.5">
         <AlertTriangle size={12} className="text-[var(--color-warning)]" />
         <span className="text-xs text-[var(--color-warning)]">
           Bound agent no longer exists. Please select a different agent.
         </span>
       </div>
     )}
     {!boundAgentId && (
       <p className="text-[10px] text-[var(--color-text-muted)] mt-1">
         Choose which agent handles messages from this Telegram bot
       </p>
     )}
   </div>
   ```

6. **Update saveTelegramConfig()** to also write the binding. Replace the existing `patchConfig({ channels: { telegram: telegramConfig } })` call with a single patchConfig that writes both channels and bindings at the config root level:
   ```typescript
   const otherBindings = allBindings.filter(b => b.match?.channel !== "telegram");
   const updatedBindings = boundAgentId
     ? [...otherBindings, { agentId: boundAgentId, match: { channel: "telegram" } }]
     : otherBindings;

   await patchConfig({
     channels: { telegram: telegramConfig },
     bindings: updatedBindings,
   });
   ```
   This is a single patchConfig call that writes both `channels.telegram` and `bindings` at the config root level. CRITICAL: `bindings` is at the config root, NOT under `channels`. The merge-patch replaces the entire `bindings` array, so always preserve non-telegram entries by filtering and re-adding.

7. **After save and reconnect**, update `allBindings` state from the reloaded config (already handled by calling `loadTelegramConfig()` which now reads bindings).
