Create `src/api/config-helpers.ts` with two exported functions:

1. `patchConfig(patch: Record<string, unknown>, options?: { restartDelayMs?: number }): Promise<{ restart: { ok: boolean; delayMs: number; coalesced: boolean } }>`
   - Calls `gateway.request<ConfigGetResponse>("config.get", {})` to get current hash
   - Calls `gateway.request<ConfigPatchResponse>("config.patch", { baseHash: hash, raw: JSON.stringify(patch), ...options })` to send partial config

   - Poll `gateway.connected` every `pollMs` until true or `maxWaitMs` exceeded
   - Return `true` if reconnected, `false` if timed out
   - On timeout, log `console.warn("[config] reconnect timeout, gateway may still be restarting")` but do NOT throw

**src/stores/settings-store.ts (8 call sites):**

Add import: `import { patchConfig, waitForReconnect } from "../api/config-helpers";`

For each method, replace the `readConfigFile()` + spread-merge + `writeConfigAndRestart()` pattern with a targeted `patchConfig()` call. Keep the `loadConfig()` and `loadModels()` calls AFTER `waitForReconnect()`.

1. `addCustomModel` — patch: `{ models: { providers: { [providerKey]: providerEntry } }, agents: { defaults: { models: { [modelKey]: {} } } } }`
2. `updateCustomModel` — patch: `{ models: { providers: { [providerKey]: updatedEntry } } }` (rebuild full provider entry since models array needs complete replacement per Pitfall 2)
3. `removeCustomModel` — patch: `{ models: { providers: { [providerKey]: null } }, agents: { defaults: { models: { [modelKey]: null } } } }` (null = delete in merge-patch). If removing one model from a multi-model provider, rebuild the provider with remaining models instead.
4. `replaceProviderModels` — patch: `{ models: { providers: { [providerKey]: updatedProvider } } }`
5. `setProviderAuth` — patch: `{ models: { providers: { [key]: providerEntry } }, agents: { defaults: { models: modelEntries } } }` (see RESEARCH.md migration example)
6. `removeProvider` — patch: `{ models: { providers: { [key]: null } } }` plus null-out each model key in `agents.defaults.models`
7. `addQuickProvider` — patch with all providers from the quick config preset
8. `removeQuickProvider` — patch with null for each provider key being removed

IMPORTANT for removal methods (3, 6, 8): Use `null` values to delete keys via merge-patch semantics (RFC 7396). For example: `{ models: { providers: { "my-provider": null } } }` removes that provider key.

IMPORTANT for `removeCustomModel` (3): This method needs to handle the case where a provider has multiple models. Read the existing implementation carefully:
- If the provider has only 1 model, set provider to null (delete it)
- If the provider has multiple models, you still need to read the current config to know what models exist. Use `gateway.request("config.get", {})` to read, filter out the model, then patch with the rebuilt provider. This is one case where a read-before-patch is necessary.

After each `patchConfig()` + `waitForReconnect()`, call `get().loadConfig()` and `get().loadModels()` as the existing code does.
