- Enable/disable toggle: a small toggle switch or button. On toggle, call `patchConfig({ channels: { telegram: { accounts: { [accountId]: { enabled: !config.enabled } } } } })` then `waitForReconnect()` then `onReload()`. This is the minimal write pattern per research (Pitfall 5).
- Click anywhere on card header (except toggle) calls `onToggleExpand`
- ChevronDown/ChevronUp icon to indicate expandability

- Rendered below compact view with `border-t` separator
- Contains all the form fields currently in IMChannelsSection's "Config Form" section:

  - Group Policy select (open/allowlist/disabled)
  - Allowed Groups via TagInput (shown when groupPolicy is allowlist)
  - Group Sender Allow-List via TagInput (shown when groupPolicy is allowlist)
- Save button: builds config object same as current `saveTelegramConfig`, calls `patchConfig` with `channels.telegram.accounts.[accountId]` + updated bindings, then `waitForReconnect()` + `onReload()`
- Remove button (red, bottom of expanded view): calls `onRemove()` prop (parent handles confirmation dialog)
- Save feedback message
