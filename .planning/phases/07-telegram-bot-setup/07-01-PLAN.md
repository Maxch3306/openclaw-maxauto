  <files>src/components/settings/IMChannelsSection.tsx</files>
  <action>
Enhance the bot token input section in IMChannelsSection.tsx to validate tokens before saving. This is an incremental change -- keep all existing DM/group policy and pairing functionality intact.

1. **Add validation state:**
   - `validating: boolean` -- whether a validation request is in progress
   - `validationResult: { valid: boolean; botUsername?: string; error?: string } | null` -- result of last validation

2. **Create `validateBotToken(token: string)` function** that:
   - First does a format pre-check: `/^\d+:[A-Za-z0-9_-]{30,}$/.test(token.trim())`. If fails, return `{ valid: false, error: "Invalid token format. Expected: 123456789:ABCdef..." }`
   - Then calls `fetch(\`https://api.telegram.org/bot${token.trim()}/getMe\`)` directly (no CORS issues with Telegram API)
   - On success (res.ok and data.ok): return `{ valid: true, botUsername: data.result?.username }`
   - On API error: return `{ valid: false, error: data?.description ?? "Invalid bot token" }`
   - On network error: return `{ valid: false, error: "Could not reach Telegram API. Check your network." }`

3. **Replace the current "Enable Telegram" / "Update" button behavior:**
   - The save button should now say "Validate & Save" (or "Validate & Update" when already configured)
   - On click: run `validateBotToken()` first. If valid, auto-save via the existing `saveTelegramConfig()`. If invalid, show error and do NOT save.
   - Show a spinner or "Validating..." text during the API call
   - After successful validation + save, show the bot username: e.g., "Connected as @BotUsername"

4. **Add validation feedback UI below the bot token input:**
   - While validating: show a subtle loading indicator text
   - On validation success: show green text with bot username (e.g., checkmark + "@YourBot")
   - On validation failure: show red text with the error message
   - Clear validation result when the user changes the token input

5. **Skip validation if token unchanged:** If `botToken` matches `telegramCfg.botToken` (already saved), skip the getMe call and save directly (the token was already validated on first entry).

Keep the existing Disable button behavior unchanged. Keep all existing imports and other sections (DM policy, group policy, pairing) untouched.
